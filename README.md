# GraphRAG-Loop

> 把 Agent Loop 用在检索上的多模态知识图谱 RAG 系统。
> 核心:知识图谱 + 逐跳 BFS/Beam 检索 + Agent 自主决策 + Self-RAG 反思,三者深度融合。

---

## 0. 一句话定位

传统 GraphRAG 是「一把梭」式子图提取,Agentic RAG 是「粗粒度」策略切换。本项目把二者结合,在**图内部做细粒度的逐跳路径选择**,每一跳都由 Agent 决策 + Self-RAG 反思来驱动「往深走 / 换方向 / 停下回答」。

```
构建知识图谱 → 每跳 Beam 扩展 → rank/剪枝 → 喂给 Agent 选 → 反思 → 下一轮 / 终止生成
```

---

## 1. 学术坐标(我们站在哪些工作的肩膀上)

| 工作 | 我们借鉴的核心机制 | 在本项目中的位置 |
|------|------------------|----------------|
| **Think-on-Graph (ToG)** [arxiv 2307.07697] | LLM-as-agent 在 KG 上做 **beam search**,逐跳剪枝关系/实体,自主判断停止 | 检索主循环的骨架 |
| **HippoRAG / HippoRAG 2** [arxiv 2502.14802] | **向量软链接多种子 + Personalized PageRank** 扩散替代脆弱的精确实体链接 | 入口召回 + 防图爆炸 |
| **Self-RAG** [arxiv 2310.11511] | 反思 token `Retrieve / IsRel / IsSup / IsUse`,按需检索 + 段级证据验证 | 三层反思注入 |
| **FLARE** | 生成中监控 token 置信度,主动触发检索 | 进阶版终止/回扩触发器 |
| **Chain-of-Verification (CoVe)** | 生成后自列验证问题、独立核验、修订 | 最终输出前的把关(可选) |

> 这套组合(Self-RAG 的反思能力来决定图遍历路径深浅与停止)已被综述明确点名为有效方向,我们做的是把它工程化 + 多模态化 + 在若干环节做原创优化。

## 2. 整体架构与数据流(带超参 / 技术选型)

```
                              ┌─────────────────────────────────────────┐
   用户 Query ───────────────▶│  L1 入口决策 (Self-RAG: Retrieve Token)   │
                              │  ─ 这个问题需要走图谱吗?                  │
                              │    否 → 直接 LLM 生成 (省掉整条检索链)      │
                              │    是 → 进入混合入口召回                    │
                              └───────────────┬───────────────────────────┘
                                              │  yes
                  ┌───────────────────────────┴────────────────────────────┐
                  │  L1b 混合入口召回 (多种子,避免实体链接单点故障)          │
                  │   ① 向量检索 top-k 实体/passage   (Embedding + ANN)      │
                  │   ② NER + 向量软链接 → 候选节点    (HippoRAG 式软链接)    │
                  │   ③ 取并集,带权重作为种子        seed_weight ∈ (0,1]    │
                  │   ─ 可选: 一次 PPR 扩散预热,给所有节点先验分            │
                  │       超参: PPR_alpha=0.5, PPR_topk=候选截断             │
                  └───────────────────────────┬────────────────────────────┘
                                              │  seeds + 先验分
                  ┌───────────────────────────▼────────────────────────────┐
                  │           ╔══════════ Agent 检索主循环 ══════════╗        │
                  │           ║  (round r = 1 .. D_max)              ║        │
                  │           ║                                      ║        │
                  │  ┌────────╨─────────┐                            ║        │
                  │  │ S1 扩展当前层邻居 │  ← Beam: 每个 frontier      ║        │
                  │  │   (Neo4j 邻居查询)│     节点取邻居,候选池        ║        │
                  │  └────────┬─────────┘                            ║        │
                  │           ▼                                      ║        │
                  │  ┌──────────────────────────────┐               ║        │
                  │  │ S2 关系剪枝 (Relation Prune)   │  ToG 式:先选边 ║        │
                  │  │   LLM/打分给关系类型排序        │  再选实体      ║        │
                  │  └────────┬───────────────────────┘               ║        │
                  │           ▼                                      ║        │
                  │  ┌──────────────────────────────┐               ║        │
                  │  │ S3 节点重排 (Rank)             │  Cross-Encoder ║        │
                  │  │   / IsRel 打分,逐节点判相关性  │  或 NLI 模型   ║        │
                  │  │   score < τ_rel → 剪掉         │  (轻量模型)    ║        │
                  │  └────────┬───────────────────────┘               ║        │
                  │           ▼                                      ║        │
                  │  ┌──────────────────────────────┐               ║        │
                  │  │ S4 Beam 截断                   │  保留 top-N    ║        │
                  │  │   topN 进下一轮 frontier        │  beam_width=N ║        │
                  │  │   记录已访问边 (路径记忆,去重)  │               ║        │
                  │  └────────┬───────────────────────┘               ║        │
                  │           ▼                                      ║        │
                  │  ┌──────────────────────────────┐               ║        │
                  │  │ S5 Agent 决策 (Self-RAG)       │               ║        │
                  │  │   累积证据置信度 conf 估计      │               ║        │
                  │  │   conf ≥ θ_stop  → 跳出,去生成  │  双约束终止    ║        │
                  │  │   r ≥ D_max      → 跳出 (兜底)  │               ║        │
                  │  │   否则 → 下一轮 (可并行多方向)  │               ║        │
                  │           ╚══════════════════════════════════════╝        │
                  └───────────────────────────┬────────────────────────────┘
                                              │  证据子图 + 路径
                  ┌───────────────────────────▼────────────────────────────┐
                  │  L3 生成 + 验证 (Self-RAG: IsSup / IsUse Token)          │
                  │   ① 基于证据子图生成带 inline 引用的答案                  │
                  │   ② 段级 IsSup 验证 (NLI 蕴含模型: passage ⊨ claim)      │
                  │       fully / partially / no support,阈值 τ_sup         │
                  │   ③ 存在 unsupported 段 → 定向回扩 (带着缺口回到主循环)   │
                  │       回扩次数 ≤ V_max (兜底,防无限打转)                 │
                  │   ④ 全部 supported → 输出 (可选 CoVe 二次核验)            │
                  └─────────────────────────────────────────────────────────┘
```

### 2.1 关键超参一览

| 符号 | 含义 | 建议初值 | 调优方向 |
|------|------|---------|---------|
| `D_max` | 最大跳数 / 循环轮次兜底 | 3 | 多跳数据集可加大 |
| `beam_width N` | 每跳保留的节点数 | 5 | 图度数大时调小 |
| `τ_rel` | IsRel/重排剪枝阈值 | 0.5 | 召回不足调低 |
| `θ_stop` | 终止置信度阈值 | 0.7 | 偏保守调高 |
| `τ_sup` | IsSup 支撑阈值 | 0.6 | 抗幻觉调高 |
| `V_max` | 定向回扩最大次数 | 2 | 延迟敏感调小 |
| `PPR_alpha` | PageRank restart 概率 | 0.5 | HippoRAG 默认 |
| `seed_topk` | 入口候选种子数 | 5~10 | 宁多勿漏 |

### 2.2 模型分工(成本/延迟分层)

| 环节 | 模型档位 | 选型建议 | 理由 |
|------|---------|---------|------|
| L1 入口决策 | 小模型 | Qwen-7B / 路由分类器 | 高频、判断简单 |
| S2 关系剪枝 | 小模型 | LLM prompt / 打分模型 | 候选多,要便宜 |
| S3 IsRel 重排 | 轻量打分模型 | **Cross-Encoder (bge-reranker)** 或 NLI | 可量化、出分数、设阈值 |
| S5 Agent 决策 | 中/大模型 | 主力 LLM | 需要推理 |
| L3 生成 | 大模型 | 主力 LLM | 质量优先 |
| L3 IsSup 验证 | NLI 模型 | **轻量蕴含模型** + 大模型兜底 | 便宜可量化 |

## 3. 由浅入深:四阶段落地路线

> 原则:每一阶段都能独立跑通、可验证,再叠下一层。不要一上来就全家桶。

### Phase 0 — 最小闭环 Demo(`demo/` 目录)
**目标**:用内存里的玩具图 + mock/小模型,把「Beam 扩展 → 剪枝 → Agent 决策 → 终止」这条 loop 逻辑跑通,不依赖 Neo4j、不依赖大模型。
- 图:手写 dict / networkx 内存图
- 排序:句向量 cosine 或直接 mock 打分
- Agent 决策:规则模拟(置信度阈值)
- 产出:验证循环控制流、终止条件、路径去重是否正确

### Phase 1 — 单模态文本 GraphRAG(`graphrag_loop/`)
**目标**:接真组件,跑通文本场景的完整链路。
- 图谱:Neo4j(LLM 抽实体关系建图)
- 入口:向量检索(Qdrant/Chroma)+ 软链接多种子
- 主循环:真 Beam + Cross-Encoder 重排 + LLM Agent 决策
- 生成:LLM + 段级 IsSup(NLI)

### Phase 2 — 注入完整 Self-RAG 三层反思
**目标**:把 L1 Retrieve、S3 IsRel、L3 IsSup 全部接上,加 PPR 入口预热、定向回扩。
- 加入 FLARE 式生成中触发(可选)
- 加入 CoVe 最终核验(可选)
- 加缓存:高频子图模式缓存

### Phase 3 — 多模态扩展
**目标**:接入图片/视频/音频,跨三层存储协同检索。
- 对象存储(MinIO)存原文件
- CLIP/Whisper 提多模态 embedding → 向量库
- Neo4j 存「文件-描述-实体」关系
- Agent 跨存储编排:Neo4j 找关系路径 → 向量库找语义 → 对象存储拉原文件

## 4. 多模态存储架构(Phase 3)

> 核心原则:多库协同,各司其职。**Neo4j 是关系推理引擎,不是万能存储**。

| 数据类型 | 主存储 | 辅助存储 | 说明 |
|---------|--------|---------|------|
| 文本 | Neo4j(实体关系)+ 向量库(语义) | Elasticsearch(关键词) | 核心数据 |
| JSON/Schema | Neo4j(若描述实体关系)| PostgreSQL JSONB / MongoDB | 配置日志类用文档库 |
| 图片/视频/音频 | 对象存储(原文件)+ 向量库(embedding)| Neo4j(元数据关系) | 三层架构,Neo4j 只存「谁和谁有关」 |

处理流程:
1. **特征提取**:图片→CLIP/ViT,音频→Whisper,视频→抽帧+CLIP
2. **向量入库**:embedding 入向量库,支持跨模态检索(文搜图)
3. **元数据关联**:Neo4j 建「图片A 拍摄于 地点B,含 人物C」关系图

## 5. 工程难点与对策(已踩过的坑)

| 难点 | 本质 | 对策 |
|------|------|------|
| **入口实体链接单点故障** | mention→节点 的桥,链错则全程在错子图打转(NER错/歧义/NIL/别名) | **不做精确链接**,改向量软链接多种子 + PPR 扩散平滑噪声 |
| **BFS 图爆炸** | 高度数节点一跳产生海量候选 | Beam 宽度 N + PPR top-p 截断 + 关系剪枝 |
| **循环不终止** | Agent 无明确停止标准 | 双约束:`θ_stop` 置信度 **或** `D_max` 轮次,先到为准 |
| **幻觉累积** | 长循环里证据漂移 | 段级 IsSup 验证 + 定向回扩(≤V_max) |
| **延迟叠加** | 每跳都加反思 LLM 调用 | 模型分层:小模型剪枝、NLI 打分、大模型只做决策/生成 |
| **图谱构建成本** | 比传统 RAG 高 2-3x | 增量更新 + 实体对齐 + 高频子图缓存 |
